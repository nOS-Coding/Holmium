What

Your personal AI OS

Holmium is Arch Linux + OpenRC tuned for local LLM inference. It boots into a TUI assistant connected to a vLLM backend running on your GPU. The core model is Qwen3.6-35B-A3B-AWQ — a 35-billion-parameter MoE model with AWQ quantization, giving you GPT-4-class reasoning on consumer GPUs.

▲

Qwen3.6-35B-A3B-AWQ

35B-param MoE model, AWQ-quantized. 3.6B active params per token. Runs on 6 GB+ VRAM via vLLM with CUDA or ROCm. GPT-4-class reasoning on your own GPU.

◆

153 tools, 35 categories

Code generation, web research, email, file ops, finance, GitHub, containers, NAS, vision, voice, image gen (FLUX.1), video gen (HunyuanVideo), and more — all local.

▬

Multi-modal by default

Vision (image analysis), FLUX.1 image generation, HunyuanVideo generation, Whisper STT (large-v3), Kokoro TTS (am_michael voice). One system, all modalities.

◇

Companion apps

Native macOS (AppKit), Windows (WPF), and Android (Material You) apps with Chat, Dashboard, and Settings. Linux uses the TUI on tty1. All connect via WireGuard.

✕

Zero telemetry

No analytics, no tracking, no logging of prompts. Only daily license check transmits your hashed machine ID — nothing else.

↺

Free updates

All updates are free while your subscription is active. Rebuild your ISO anytime. Cancel and keep access until end of billing period.

◯

NAS-ready

Built-in WireGuard VPN, Samba/NFS exports, and automated backup tooling. Turn your AI PC into a home server when it's not running models.

▩

Dual-boot + headless

Install alongside Debian, Ubuntu, Fedora, KDE Neon, Nobara, or CachyOS. Or run without a monitor — SSH in from any companion app.

Downloads

Download an ISO

Each variant is a bootable ISO. Flash to USB with dd and boot.

⚠ DANGER — READ BEFORE INSTALLING

Installing Holmium will wipe the target drive completely. The installer repartitions and formats the entire disk. There is no "keep files" option. Back up everything you care about before booting the ISO. Dual-booting is supported (Holmium detects existing OSes), but the target disk itself will be erased.

FAQ

Common questions

  • The core model is Qwen3.6-35B-A3B-AWQ — a 35-billion-parameter Mixture-of-Experts model by Qwen, AWQ-quantized for consumer GPUs. Only 3.6B parameters activate per token, giving GPT-4-class performance in 6 GB+ VRAM. It also uses FLUX.1 for image generation, HunyuanVideo for video, Whisper large-v3 for speech-to-text, and Kokoro (am_michael) for text-to-speech.
  • Yes. The installer repartitions and formats the entire target drive. All existing data on that disk will be permanently lost. If using dual-boot, you keep your other OS partitions (detected automatically), but the Holmium target disk is erased. Always back up your data before installing.
  • Installing an operating system always carries risk. Holmium is beta software. Potential issues include: data loss (full disk erase), bootloader conflicts with existing OSes, GPU driver incompatibility on unsupported hardware, and network configuration issues on unusual setups. Holmium targets NVIDIA RTX 5060–5090 and AMD RX 9060 XT–9070 XT GPUs. Anything else may not work or may cause instability. Back up all data and have a recovery USB ready.
  • Minimum: NVIDIA RTX 5060 (6 GB VRAM) or AMD RX 9060 XT (8 GB VRAM), 16 GB system RAM, 512 GB storage. Recommended: RTX 5070+ or RX 9070 XT, 32 GB RAM, 1 TB NVMe. Integrated GPUs will not work.
  • Yes. Holmium supports dual-boot with Debian, KDE Neon, Ubuntu, Fedora, Nobara, and CachyOS. The installer auto-detects existing OSes and sets up GRUB chain-loading.
  • The ISO boots and you can explore, but the Holmium backend (inference, assistant, tools) requires an active subscription. There is no trial period.
  • Your license key is Ed25519-signed and bound to /etc/machine-id. Subscribe via Lemon Squeezy, get a key. It validates locally with daily online checks. One license per machine.
  • Yes, anytime. Your license stays active until the end of the billing period. After that, inference stops. No fees, no lock-in, no data loss.
  • The companion apps, build scripts, and dual-boot detection are on GitHub. The backend and model serving stack are proprietary.
  • AWQ (Activation-aware Weight Quantization) with group-size 128 for 6 GB+ GPUs and group-size 64 for 16 GB+ GPUs. This gives the best quality-to-speed ratio for local inference.
